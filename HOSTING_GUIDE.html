Path A · Netlify (recommended)
1
Publish instantly
  1. Go to app.netlify.com/drop.
  2. Drag the ZIP file itself (the .zip you downloaded) straight onto the page — Netlify unpacks it for you. This is the most reliable method.
  3. Done — it is live at a free random-name.netlify.app address.
Prefer to unzip first? Then open the unzipped folder so you can see index.html, select all the files inside (Ctrl/Cmd + A), and drag those files onto Netlify — not the parent folder. index.html must sit at the top level of what you drop.
Create a free account to keep the site, rename it, and add a custom domain.
⚠️
If you see “Page not found” (404)
This almost always means index.html was not at the top level of what you uploaded — usually because an extra wrapper folder got dragged in (Netlify ended up serving a folder that just contains your site folder).
  • Fix 1 (easiest): redo the deploy and drag the .zip file itself onto app.netlify.com/drop. The zip is packaged so every file sits at the root.
  • Fix 2: if you unzipped, open the folder, press Ctrl/Cmd + A to select all files, and drag the files (you should see index.html among them) — not the folder.
  • Check: in Netlify go to your site → Deploys → click the latest deploy → “Browse files”. You must see index.html listed at the top. If instead you see a single folder, that is the problem — re-deploy using Fix 1.
This package includes a netlify.toml that pins the site root, plus a branded 404.html, so when the upload is correct everything resolves properly.
2
Update it later
Edit your files locally, then drag the folder onto Netlify again (or connect it to GitHub for automatic updates on every save). Redeploys in seconds.
Path B · Cloudflare Pages
1
Deploy
  1. Sign up at pages.cloudflare.com.
  2. Choose Direct Upload and upload your folder (or connect a GitHub repo).
  3. Live on a .pages.dev address with free SSL.
Path C · GitHub Pages
1
Deploy from a repo
  1. Create a free repo at github.com and upload all files.
  2. Settings → Pages → Deploy from branch main, folder / (root).
  3. Live at yourname.github.io/repo.
Your own domain & SSL
🌐
Point fintexglobal.com at your site
  1. Buy/own the domain (e.g. via Cloudflare Registrar — at cost — or any registrar).
  2. In your host (Netlify/Cloudflare Pages) open Domains → Add custom domain and enter fintexglobal.com.
  3. Update your domain's DNS records as the host instructs (they show you exactly what to paste).
  4. SSL is issued automatically and free — your site gets the padlock and https://.
DNS changes can take a little while to take effect (minutes to a few hours). That is normal.

Ongoing maintenance
  • Re-deploy whenever you edit content (drag folder / git push).
  • Banner/testimonial edits made in the portal show on your own device immediately; to show them to all visitors, either copy the change into index.html and re-deploy, or connect a backend so the homepage reads them live (see WEBSITE_GUIDE.html → “Publishing content live”).
  • Onboarding form changes (Portal → Onboarding Form) are stored in your browser like the content edits above; connect a backend to share the same form live across every device. Note: the form never collects TFN/ABN — that is intentional and required (see WEBSITE_GUIDE / INTEGRATION_GUIDE).
  • Check Leads, Sign-up Approvals and reports in the portal regularly.
  • Renew your domain yearly (set a calendar reminder).
  • When data needs to be shared across your team, follow the backend steps in the Integration Guide.
