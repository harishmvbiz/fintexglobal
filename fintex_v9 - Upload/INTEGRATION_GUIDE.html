📊 1. Google Sheets Free

Every contact form lead and client portal submission gets auto-saved to a Google Sheet. No backend server required — uses Google Apps Script webhook.

Step 1 — Create the Sheet

  1. Go to sheets.google.com → Create new spreadsheet
  2. Name it: Fintex CRM
  3. Create two sheets (tabs): Leads and Submissions
  4. In Leads sheet, add headers in row 1: Date | Name | Email | Phone | Region | Service | Message
  5. In Submissions sheet, add: Date | Client | Company | Region | Service | Period | Revenue | Files | Notes

Step 2 — Apps Script Webhook

In your Google Sheet: Extensions → Apps Script → paste this code:

function doPost(e) {
  try {
    var data = JSON.parse(e.postData.contents);
    var ss = SpreadsheetApp.getActiveSpreadsheet();
    
    if (data.type === 'lead') {
      var sheet = ss.getSheetByName('Leads');
      sheet.appendRow([
        new Date().toLocaleString(),
        data.name, data.email, data.phone,
        data.region, data.service, data.message
      ]);
    }
    
    if (data.type === 'portal_submission') {
      var sheet = ss.getSheetByName('Submissions');
      sheet.appendRow([
        new Date().toLocaleString(),
        data.client, data.company, data.region,
        data.service, data.period, data.revenue,
        (data.files || []).join(', '), data.notes
      ]);
    }
    
    return ContentService
      .createTextOutput(JSON.stringify({status: 'ok'}))
      .setMimeType(ContentService.MimeType.JSON);
      
  } catch(err) {
    return ContentService
      .createTextOutput(JSON.stringify({status: 'error', msg: err.toString()}))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
  1. Click Deploy → New deployment
  2. Type: Web app
  3. Execute as: Me
  4. Who has access: Anyone
  5. Click Deploy → Authorize → Copy the Web App URL

Step 3 — Connect to Portal

  1. Go to Admin Portal → Notifications
  2. Paste the Web App URL in Google Sheet Webhook URL
  3. Save. All future leads and submissions auto-save to your sheet ✓
💡 Tip: Share the Google Sheet with arun@tensquare.com.au so both admins see live data.

📧 2. EmailJS — Email Notifications Free (200/mo)

Send yourself an email whenever a lead comes in or a client submits their portal. No backend server needed.

  1. Sign up at emailjs.com (free — 200 emails/month)
  2. Add Email Service: connect your Gmail (info@fintexglobal.com)
  3. Create Email Template — click Email Templates → Create Template
  4. Use this template body:
Subject: 🔔 New {{submission_type}} — Fintex Portal

A new {{submission_type}} has been received:

Client: {{client_name}}
Email: {{client_email}}
Region: {{region}}
Service: {{service}}

  1. Save template. Copy Service ID, Template ID, and Public Key
  2. Paste all three into Admin Portal → Notifications

💬 3. WhatsApp Notifications

Two options depending on budget and volume:

Option A — WhatsApp link (Free, manual)

When a client submits, the portal generates a pre-filled WhatsApp link. Click it to open WhatsApp with submission details pre-typed.

This works automatically — no setup needed. The link uses your number 971557282794.

Option B — Twilio WhatsApp Sandbox ~$0.005/msg

  1. Sign up at twilio.com
  2. Go to Messaging → Try WhatsApp Sandbox
  3. Send the join code from your WhatsApp to activate sandbox
  4. Get Account SID and Auth Token from dashboard

☁️ 4. Google Drive — Document Storage Free (15GB)

Auto-upload client documents from the portal directly into a structured Google Drive folder.

Simple Setup (Manual, no API)

  1. Create folder in Drive: Fintex-Clients/
  2. Inside, create sub-folders per client: James-Whitfield/, Priya-Sharma/
  3. When clients submit, download files from portal and upload to their Drive folder
  4. Share the folder with arun@tensquare.com.au for joint access

Automated Setup (Drive API) Free

  1. Go to console.cloud.google.com → Create Project: Fintex Portal
  2. Enable Google Drive API
  3. Create Service Account → Download JSON key file
  4. Create folder Fintex-Client-Docs in Drive → Share with service account email
  5. Deploy a small serverless function (Netlify Function / Cloudflare Worker) to proxy the upload: 
    // netlify/functions/upload-to-drive.js
const { google } = require('googleapis');

exports.handler = async (event) => {
  const auth = new google.auth.GoogleAuth({
    credentials: JSON.parse(process.env.GOOGLE_SERVICE_KEY),
    scopes: ['https://www.googleapis.com/auth/drive']
  });
  const drive = google.drive({ version: 'v3', auth });
  const { fileName, fileData, clientName } = JSON.parse(event.body);
  
  // Find or create client folder
  const folder = await drive.files.create({
    requestBody: { name: clientName, mimeType: 'application/vnd.google-apps.folder',
      parents: [process.env.FINTEX_FOLDER_ID] }
  });
  
  // Upload file
  await drive.files.create({
    requestBody: { name: fileName, parents: [folder.data.id] },
    media: { body: Buffer.from(fileData, 'base64') }
  });
  
  return { statusCode: 200, body: JSON.stringify({ ok: true }) };
};
  6. Set environment variables in Netlify: GOOGLE_SERVICE_KEY, FINTEX_FOLDER_ID

🟦 5. Microsoft OneDrive Microsoft 365 required

  1. Register app at portal.azure.com → Azure Active Directory → App Registrations → New Registration
  2. Name: Fintex Portal → Accounts in this org only → Register
  3. API Permissions → Add → Microsoft Graph → Files.ReadWrite.All → Grant admin consent
  4. Certificates & Secrets → New client secret → Copy value immediately
  5. Note: Application (client) ID and Directory (tenant) ID
  6. Create folder in OneDrive: /Fintex-Clients/
  7. Get folder ID via Graph Explorer: GET /me/drive/root:/Fintex-Clients
  8. Use Microsoft Graph API to upload files: 
    // Get token
const token = await fetch(
  `https://login.microsoftonline.com/${TENANT_ID}/oauth2/v2.0/token`,
  { method: 'POST', body: new URLSearchParams({
    client_id: CLIENT_ID, client_secret: CLIENT_SECRET,
    grant_type: 'client_credentials',
    scope: 'https://graph.microsoft.com/.default'
  })}
).then(r=>r.json());

// Upload file
await fetch(
  `https://graph.microsoft.com/v1.0/drives/${DRIVE_ID}/items/${FOLDER_ID}:/${fileName}:/content`,
  { method: 'PUT', headers: { Authorization: 'Bearer '+token.access_token }, body: fileBuffer }

📁 6. FYI Document Manager ~$49/mo

FYI is purpose-built for accounting firms — integrates with Xero, MYOB, QBO and has smart client folder structure.

  1. Sign up at usefyi.com → Start free trial
  2. Settings → Integrations → API → Generate API Key
  3. Create clients in FYI matching names in the Fintex portal
  4. Use FYI API to upload: 
    // POST /api/v1/documents
fetch('https://api.usefyi.com/api/v1/documents', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_FYI_API_KEY',
    'Content-Type': 'multipart/form-data'
  },
  body: formData  // FormData with file + client_id + folder
});
  5. Files automatically appear in client's FYI folder, linked to their contact record
💡 Best for: When you have 10+ active clients and want proper version control, email integration, and workflow automations.
